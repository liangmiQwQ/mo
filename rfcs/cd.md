# RFC: `cd` command

`mo cd` opens an interactive selector that lets the user navigate to any repo, owner directory, or the code root. Because a child process cannot change the parent shell's working directory, the command is implemented as a shell function.

## Usage

```bash
mo cd
```

## Shell Integration

The shell rc block generated by `mo setup` wraps the `mo` binary with a shell function. When `mo cd` is invoked:

1. The shell function calls `mo-inner cd`, which prints the resolved path to stdout (from `MO_CD_TARGET` env var, or `.` when unset).
2. The shell function runs `cd` on that path.

This means `mo cd` itself must never be called directly as a plain binary invocation for the directory change to take effect.

## Selector UI

The interactive selector is specified in [selector.md](./selector.md). Key points:

- Displays `<root>`, owner directories, and individual repos in grouped list mode.
- Supports incremental search across repo names and owner names.
- Arrow keys navigate; Enter confirms; Esc / Ctrl-C cancels.

## Selectable Targets

| Target           | Path                    |
| ---------------- | ----------------------- |
| `<root>`         | `~/code`                |
| `<owner>`        | `~/code/<owner>`        |
| `<owner>/<repo>` | `~/code/<owner>/<repo>` |
